# 逐页改造 Prompt（赛博朋克 / 边缘行者风）— 给 Claude Design

> 用法：先把 `DESIGN_BRIEF.md` 整段给 Claude Design 建立全局风格与设计系统；再逐页把下面对应段落 + 该页截图（`docs/design_shots/` 内同名 PNG）一起贴进去。
> 每段格式：**保留**（别破坏的结构/信息）｜**霓虹化**（要施加的赛博朋克处理）。
> 全局基调：近黑底（#0a0a0f）+ 霓虹青(#00f0ff)/品红(#ff2a6d)/电黄(#fcee0a)/紫(#bd00ff)；辉光、扫描线、HUD 框线、故障感、单色等宽科技标题字；中文需单独配一款科技感且支持中文的字体。状态色映射：成功=青绿、运行中=青、待审批=电黄、失败=品红/红、暂停=橙。

---

## 全局外壳 · 顶部导航 GlobalNav
**保留**：左品牌「Novel Local AI」、中部三入口（项目/运行记录/设置）、运行记录待审批红点、右「本地数据·SQLite」徽标。
**霓虹化**：导航条做成 HUD 状态栏——半透明深黑玻璃 + 底部 1px 青色发光线；品牌名用等宽科技字 + 轻微故障/扫描质感；激活项下方换成青色霓虹流光下划线；待审批红点换成品红呼吸闪烁的菱形/警示徽标；右侧「本地数据·SQLite」做成带小绿点的终端式状态胶囊（像系统在线指示）。

## 01 首页/项目列表（截图 01_dashboard）
**保留**：大标题「分析、解构、仿写小说」、上传分析/新建两个主操作、四宫格统计（待审批/运行中/失败/可用模型）、三列项目卡片网格、空状态。
**霓虹化**：四宫格统计卡做成 HUD 数据面板——深黑卡 + 霓虹描边 + 大号发光数字 + 角标式 L 形 HUD 边框，数字按语义色发光（待审批电黄、运行中青、失败品红、模型青绿）；项目卡换成"义体档案卡"：深色玻璃、青/品红描边、hover 时霓虹辉光抬起 + 轻微扫描线扫过；主标题用科技展示字 + 渐变霓虹描边；上传按钮做成"注入数据"式的发光主按钮。

## 02 项目总览（截图 02_project_overview）
**保留**：深色英雄"下一步行动"引导卡、4 格统计条、模型就绪度卡、故事设定完整度、最近章节/最近运行两列。
**霓虹化**：英雄卡本就深色，强化成 braindance/任务简报面板——青色 HUD 边框 + 标题霓虹辉光 + 行动按钮做成发光 CTA；4 格统计做成进度型 HUD 读数（章节进度用青色分段进度条）；模型就绪度做成"系统在线/离线"指示（绿点脉冲 vs 品红告警）；最近章节/运行列表行 hover 霓虹高亮 + 状态用发光点。

## 03 ★章节工作区（截图 03_chapters_work）— 核心页
**保留**：三栏布局（左章节列表 / 中正文编辑：标题输入+目标+大纲+相关角色多选+引用搜索 / 右 Loop 配置启动面板）；这页信息密度高、是主战场，**结构不能动**。
**霓虹化**：整体像"创作驾驶舱/义体终端"。左栏章节卡做成竖排关卡列表，每张带状态发光徽标（已批/待审/仅大纲不同霓虹色）、选中态青色发光边；中栏正文编辑区像全息文档——深黑底 + 细网格 + 大标题霓虹光标感，目标/大纲文本域用 HUD 框线；相关角色多选胶囊做成发光标签芯片；右栏 Loop 配置面板做成"任务装配台"：每个下拉/数字输入是 HUD 控件，启动按钮是醒目的霓虹"START LOOP"。保持桌面三栏不塌。

## 04 创作中心（截图 04_creative_studio）
**保留**：顶部三模式切换（想法/拆解/仿写）、左输入区（大文本域+上传框+两级模型选择+拆解维度多选+进度条）、右侧栏、底部候选卡列表（逐条接受/拒绝/批量按置信度）。
**霓虹化**：三模式切换做成发光分段开关（选中态霓虹填充+辉光）；上传框做成"数据注入舱"虚线霓虹框 + hover 流光；拆解维度多选按钮做成可亮灭的霓虹标签矩阵；进度条用青色霓虹流动填充 + 百分比 HUD 读数；候选卡做成"扫描结果卡"：每条带置信度环形/条形发光读数、接受=青绿发光、拒绝=品红，批量工具条像控制台按钮。

## 05 人物卡（截图 05_characters）
**保留**：左角色列表 + 右编辑表单（角色字段 + JSON 文本域）。
**霓虹化**：左列表做成"人物数据库/通缉档案"侧栏，每个角色是带头像位的霓虹档案条；右表单做成角色 dossier——字段标签用全大写等宽小字 + 青色，输入框 HUD 框线，JSON 文本域做成终端代码窗（深黑+等宽+青色光标）；保存按钮发光。

## 06 世界观设定（截图 06_worldbuilding）
**保留**：左规则列表（带优先级徽章）+ 右编辑表单（名称/分类/优先级/描述）。
**霓虹化**：规则列表做成"世界协议/规则数据库"，每条带优先级 HUD 标记（P{n} 用电黄/品红分级发光）；分类下拉、优先级数字输入做成 HUD 控件；整体像在编辑一套不可变的系统规则（呼应"不可变世界观规则"），用锁/协议视觉语言 + 霓虹边。

## 07 Loop Runs 列表（截图 07_loop_runs）
**保留**：状态筛选胶囊、多章生产线面板、Loop Run 五列表格行。
**霓虹化**：筛选胶囊做成发光过滤标签；多章生产线面板做成"流水线监控台"——进度条霓虹流动 + 当前章节高亮 + 操作按钮像控制台；运行行做成监控日志行：状态徽标发光（运行中青/待审电黄/失败品红/完成青绿）、hover 整行霓虹扫过 + 等宽时间戳。

## 08 ★Run 详情（截图 08_run_detail）— 状态最丰富
**保留**：头部状态徽标、失败/暂停/审批多色横幅、左栏实时初稿流式预览/版本预览、右栏（自动策略卡+状态机时间线+连续性报告+修订计划）、底部 sticky 人工决策栏。
**霓虹化**：整页像"任务运行监视器"。头部状态徽标做成发光状态灯；各色横幅换成 HUD 告警条（失败=品红故障条带扫描线、暂停=橙、审批=电黄、成功=青绿）；左栏流式初稿预览做成终端打字机效果（深黑+青色流式光标+扫描线）；右栏状态机时间线做成竖向霓虹节点链（每步发光圆点+连接线，当前步脉冲）；连续性报告做成"诊断/告警面板"，每条 issue 按严重度发光；底部决策栏做成醒目的霓虹指挥条（批准=青绿发光主按钮、拒绝=品红、修订=橙）。

## 09 运行记录/活动（截图 09_activity）
**保留**：两排筛选胶囊（类型+状态）、分组折叠的记录行、彩色状态徽标。
**霓虹化**：做成"全局活动监控墙/事件流"——筛选胶囊发光、记录行像实时事件日志（等宽时间、状态发光点、类型用不同霓虹色标签）、分组展开三角做成 HUD 展开控件、hover 行霓虹高亮。

## 10 设置·本地模型（截图 10_settings_models，整页很长）
**保留**：推荐与任务分配（三角色卡）、模型配置（折叠，左列表右表单）、高级参数表、本地模型清单（扫描卡网格）、两张基准结果表（连续性检查/正文写作，含彩色数值单元格+结论徽章）。
**霓虹化**：三角色卡做成"系统模块装配"卡（正文/检查/摘要各一块，推荐项青色发光高亮 + ★推荐徽标霓虹）；模型清单卡做成"硬件/模型库存"档案卡（参数/量化/架构用 HUD 小标签、状态发光点）；两张基准表做成"测试控制台数据表"——表头等宽全大写、数值单元格按阈值发光（高=青绿、中=电黄、低=品红）、结论徽章霓虹胶囊；整体像一个本地 AI 的控制中心仪表盘。

## 11 设置·提示词（截图 11_prompts）
**保留**：双栏——左按四组列出 ~28 个提示词模板、右大号代码编辑器。
**霓虹化**：左列表做成"指令库/协议清单"，每条带等宽 key（青色）+ 版本号 HUD 标记，选中态霓虹发光；右编辑器做成全屏终端代码窗——深黑 + 等宽 + 行号 + 青色光标 + 顶部窗口控制点，保存按钮发光；占位符提示用终端注释风。

## 12 设置·应用（无独立截图，是设置内一个 tab）
**保留**：本地数据说明卡 + 「本地数据·SQLite」徽标。
**霓虹化**：做成"系统/存储状态"面板——本地数据库做成带绿点在线指示的终端式状态卡，强调"全部数据在本机"的隐私感（用锁/盾/离线视觉语言 + 霓虹边）。

## 给 Claude Design 的落地提醒
- 先产出**设计系统**（Tailwind config 颜色 token + CSS 变量 + 字体 + 4 个核心组件的霓虹版：`.panel`/`.field`/`.btn`/徽章+表格），再逐页套。
- 所有炫酷效果要能用 Tailwind + 少量自定义 CSS（box-shadow 发光、伪元素扫描线、渐变描边）落地，避免重到卡顿。
- **中文可读性优先**：霓虹/故障/低对比别伤正文阅读；正文区（章节正文、提示词、JSON）要保证清晰。
- 保持桌面三栏/双栏信息架构，别为炫酷牺牲可用性与密度。
