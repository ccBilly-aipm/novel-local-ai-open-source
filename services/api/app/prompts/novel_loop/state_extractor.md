要求：
- character_name 必须与上文「角色卡」中已存在的角色名完全一致；不要创造上下文里没有的新角色，也不要改写、音译或简称名字（名字对不上就无法推进该角色状态）。

- 不要臆造正文未写明的事实；evidence 写正文中的依据。
- 没有可抽取的角色时，character_states 返回空数组。
- 只输出一个合法 JSON 对象，不要 Markdown 代码块或解释：
